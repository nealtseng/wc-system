**System Context:**

### 💡 提示詞提煉邏輯說明（供你核對開發遺漏）：
1. **基礎設施限制可視化**：因為專案堅持「零成本」，Aiven PostgreSQL 的 1GB 限制 以及 API-Football 的每日 100 次限制 是系統生死的關鍵，必須放在首頁監控。
2. **核心演算法解構**：將 $P_{final}$ 拆解為 W1、W2、W3，能在前端幫助你直觀檢查「微觀因子的截斷函數 (Clipping)」與「宏觀盤口背離」是否正常運作。
3. **排程器與爬蟲檢驗**：特意加入「狀態機排程器（12h/2h/15m）」與「爬蟲健康度」，你可以藉由前端面板確認非同步 Goroutine 是否在對的時間點正確觸發。
4. **防禦型攻擊檢視**：加入 Fractional Kelly（分數凱利）的部位顯示，確保系統不會因為輸出過大比例而導致破產。

### 已完成的用戶故事接入

- **量化預測實驗室**（`prediction-model`）：點擊「啟動量化預測模型」→ 呼叫 `POST /api/predict`（ELO + Poisson）→ 三角形 SVG 頂點動態更新 → 點擊 MC 模擬後呼叫 `POST /api/narrative`（DeepSeek V4 Flash）→ 真實敘事文字 + 置信度 float 顯示於 Protocol Terminal。
- **資料管線監控**（`data-operations`）：`GET /api/pipeline/status` 回傳各 scraper 狀態；賽前 12h/2h/15m odds 排程由後端 scheduler 依開球時間計算；`POST /api/pipeline/sync/odds` 可手動即時更新賠率。
- **小組賽況**（`upcoming-matches`）：`GET /api/matches` + `GET /api/signals` 顯示賽程、台灣時間、賠率與 EV 信號。
